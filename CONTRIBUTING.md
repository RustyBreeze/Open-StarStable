# Contributing to Open-StarStable

Thank you for your interest in contributing. This project is in early stages 
and all kinds of help are welcome - you do not need to be familiar with Star 
Stable Online to contribute meaningfully.

---

## Before you start

Please **open an Issue** before beginning any significant work. This keeps 
efforts from overlapping and lets us discuss the approach before time is 
invested. For small things like documentation fixes, just open a pull request 
directly.

---

## What we need most right now

- **Launcher developer** - Milestone 1, no server 
  needed, see the Wiki
- **Server developer** - Protocol implementation
- **Binary analysis / patching** - Taking the static analysis further and 
  testing identified patch targets on the 2014 executable
- **Documentation** - Expanding the Wiki, file format research, VM testing

Full details on each of these are in the 
[Wiki](https://github.com/RustyBreeze/Open-StarStable/wiki).

---

## How to contribute

1. Fork the repository
2. Create a branch for your work (`git checkout -b your-feature-name`)
3. Make your changes
4. Open a pull request with a clear description of what you did and why

---

## Real-time communication

If real-time coordination is needed, it takes place in a **shared Matrix room**.

- Direct messages are not used
- All discussion should remain in shared spaces for transparency
- Access is provided only to active contributors

---

## Ground rules

- Be respectful. This is a small community project.
- No original Star Stable Online assets may be committed to this repository.
- Keep contributions GPL-2.0 compatible.
- If you're unsure whether something is in scope, open an Issue and ask first.
- This project has no affiliation with and holds no animosity toward other 
  SSO-related projects, tools, or private servers past or present. Contributors 
  are expected to maintain the same stance. Harassment of or conflict with 
  developers from other projects will not be tolerated and may result in being 
  removed as a contributor.
- Contributors may reference publicly available code from other projects for 
  inspiration or to understand how something was approached, but directly 
  copying or adapting code from other projects is not permitted without 
  explicit permission from the original author and proper attribution. 
  Everything in this repository should be written from scratch. If you 
  referenced someone else's approach to solve a problem, note it in your 
  pull request — transparency about influences is fine, copying is not.

---

## Legal reminder

This project targets only the abandoned 2014 SSO client. Contributors should 
not attempt to reverse engineer, capture traffic from, or otherwise interact 
with the modern live Star Stable Online service.
